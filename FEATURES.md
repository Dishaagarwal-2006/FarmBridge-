System Overview
FarmBridge is a full-stack web application built with modern cloud architecture. It connects farmers and buyers through a marketplace interface with real-time market data and AI-powered recommendations.

🔧 Technology Stack
Frontend

Framework: React 18+
Styling: Tailwind CSS + Custom CSS
State Management: React Hooks (useState, useContext)
HTTP Client: Fetch API
Responsive Design: Mobile-first approach

Backend

Runtime: Node.js (generated by MeDo)
Language: JavaScript
Architecture: REST API
Async Processing: Promises/Async-Await

Database

Type: Cloud-hosted (MeDo platform)
Structure: Relational (SQL-like)
Caching: In-memory for frequent queries

External APIs

data.gov.in API:

Endpoint: /resource/9ef84268-d588-465a-a308-a864a43d0070
Data: 30,000+ commodity prices
Update Frequency: Hourly
Authentication: API Key


OpenWeatherMap API:

Endpoint: /data/2.5/weather
Data: Real-time temperature, humidity, forecast
Update Frequency: Real-time
Authentication: API Key



Deployment Platform

Host: AppMeDo (Baidu MeDo Cloud)
Infrastructure: Serverless
CDN: Global edge network
SSL/TLS: Auto-configured HTTPS

Authentication & Security

Identity Verification: Aadhaar (Government of India)
Session Management: JWT tokens
Password Security: Bcrypt hashing
Phone Verification: OTP via SMS


📊 Database Schema
Users Table
sqlCREATE TABLE Users (
  id VARCHAR(36) PRIMARY KEY,
  role ENUM('farmer', 'buyer'),
  phone_number VARCHAR(10) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  aadhaar_verified BOOLEAN DEFAULT false,
  aadhaar_number VARCHAR(12),
  district VARCHAR(100),
  state VARCHAR(100),
  verification_status ENUM('pending', 'verified', 'rejected'),
  trust_score FLOAT DEFAULT 0.0,
  average_rating FLOAT DEFAULT 0.0,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);
Crop Listings Table
sqlCREATE TABLE Crop_Listings (
  id VARCHAR(36) PRIMARY KEY,
  farmer_id VARCHAR(36) NOT NULL,
  crop_name VARCHAR(100) NOT NULL,
  description TEXT,
  quantity_kg INT NOT NULL,
  price_per_kg FLOAT NOT NULL,
  location_district VARCHAR(100),
  location_state VARCHAR(100),
  harvest_date DATE,
  availability_status ENUM('available', 'sold', 'expired'),
  photos JSON,
  created_at TIMESTAMP,
  FOREIGN KEY (farmer_id) REFERENCES Users(id)
);
Mandi Prices Table (Cached)
sqlCREATE TABLE Mandi_Prices (
  id VARCHAR(36) PRIMARY KEY,
  commodity_name VARCHAR(100),
  state VARCHAR(100),
  market_name VARCHAR(100),
  min_price FLOAT,
  max_price FLOAT,
  modal_price FLOAT,
  price_date DATE,
  last_updated TIMESTAMP,
  INDEX (commodity_name, state)
);
Transactions Table
sqlCREATE TABLE Transactions (
  id VARCHAR(36) PRIMARY KEY,
  buyer_id VARCHAR(36) NOT NULL,
  farmer_id VARCHAR(36) NOT NULL,
  crop_listing_id VARCHAR(36) NOT NULL,
  quantity_ordered INT,
  total_amount FLOAT,
  status ENUM('pending', 'completed', 'cancelled', 'disputed'),
  created_at TIMESTAMP,
  completed_at TIMESTAMP,
  FOREIGN KEY (buyer_id) REFERENCES Users(id),
  FOREIGN KEY (farmer_id) REFERENCES Users(id)
);
Ratings Table
sqlCREATE TABLE Ratings (
  id VARCHAR(36) PRIMARY KEY,
  rater_id VARCHAR(36) NOT NULL,
  ratee_id VARCHAR(36) NOT NULL,
  rating FLOAT (1-5),
  review TEXT,
  transaction_id VARCHAR(36),
  created_at TIMESTAMP,
  FOREIGN KEY (transaction_id) REFERENCES Transactions(id)
);

🔄 Data Flow
Farmer Adding Crop Listing
1. Farmer fills form:
   - Crop name, quantity, price, location, photos

2. Frontend validation:
   - Check all fields filled
   - Validate price format
   - Check photo size

3. API call to Backend:
   POST /api/crops/create
   {
     crop_name, quantity, price, location,
     photos_base64, farmer_id
   }

4. Backend processing:
   - Verify farmer authentication
   - Query mandi_prices for fair price suggestion
   - Insert into Crop_Listings table
   - Return listing_id

5. Frontend shows:
   - "Crop listed successfully!"
   - Suggested price vs fair mandi price
Buyer Viewing Available Crops
1. Buyer opens "Browse Crops" page

2. Frontend makes API call:
   GET /api/crops/search?state=Rajasthan&crop=wheat

3. Backend processes:
   - Query Crop_Listings table
   - Join with Users table (farmer info)
   - Join with Ratings table (farmer ratings)
   - Return 20 results with pagination

4. Frontend displays:
   - Crop cards with photos, price, farmer rating
   - "Contact Farmer" button

5. Buyer clicks "Contact Farmer":
   - Backend logs inquiry
   - Frontend opens WhatsApp with pre-filled message
Live Mandi Prices Update
1. Every hour, scheduled task runs:
   - Call data.gov.in API
   - Fetch latest 30,000 commodity prices
   - Update Mandi_Prices table
   - Clear old cached data (>7 days)

2. When farmer/buyer views "Mandi Prices":
   - Query from Mandi_Prices cache (very fast)
   - No direct API calls needed
   - Display 30,000+ prices instantly

3. When AI Price Predictor calculates:
   - Pull current + historical prices from cache
   - Analyze trend (up/down/stable)
   - Generate recommendation
Weather Forecast Flow
1. Farmer views Dashboard

2. Backend detects farmer location (district)

3. API call to OpenWeatherMap:
   GET /weather?q={farmer_district}&appid={API_KEY}

4. Backend receives:
   - Current temp, humidity, wind speed
   - 3-day forecast

5. AI analysis:
   - If temp > 40°C: Show "Heat wave alert"
   - If humidity > 70%: Show "Fungal disease risk"
   - If rain expected: Show "Good for sowing"

6. Frontend displays crop-specific tips in Hindi
